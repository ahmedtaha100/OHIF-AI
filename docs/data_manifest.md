# Data Manifest - RL nnInteractive Phase 0

This manifest covers every data-like input used by the Phase 0 scaffold so far.
It is not a benchmark manifest, not a tumor benchmark, and it does not certify any clinical result.

## Rules

- No PHI, DICOM, private scans, or raw medical data are tracked in this repo.
- Raw volumes, checkpoints, and derived masks remain ignored by `.gitignore`.
- Synthetic masks generated in tests are allowed because they are code-created toy arrays.
- Any future real evaluation must add source-image spacing, GT provenance, license, split, and checksum before reporting metrics.

## Current Local Public Fixture Use

- Real smoke: loads `nibabel_anatomical_fixture`, runs one positive center point through the real `nnInteractive_v1.0` checkpoint, and writes ignored outputs under `artifacts/rl_nninteractive/real_smoke/`.
- Baseline public-image verification: loads the same public fixture, creates `nibabel_center_synthetic_gt`, runs the mock Gymnasium env and deterministic robot user, and writes ignored JSON under `artifacts/rl_nninteractive/baseline_verification/public_nibabel/`.
- Neither path is a tumor benchmark or a real GT evaluation.

## Required Before Any Future Real Benchmark

- Dataset name, version/date, license, and access URL or accession.
- Public/de-identified status and exclusion of PHI.
- Image modality and voxel spacing source.
- GT provenance, annotator/source, label definition, and split assignment.
- Checksums or immutable artifact IDs for images and labels.
- Exact nnInteractive checkpoint revision and inference device.
